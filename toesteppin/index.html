Legacy IEs + IE9 = toesteppin’

You should view this page in IE9.

Naïve

Originally, IE9 betas didn’t honor the legacy -ms-filter or filter code used to deliver gradients to IE. So the naïve approach was to use those properties for gradients in IE8 and earlier browsers, and use SVG or another method for gradients in IE9.

But in the IE9 RC, Microsoft have decided to bring back -ms-filter to IE9 Standards Mode (instead of just IE7 and IE8 Standards Modes).

This might be alright, except for that now CSS rounded corners are completely obliterated by -ms-filter gradients!

This div needs a gradient and rounded corners, but the corners don’t show in IE9.
(Gradient applied via -ms-filter, though also defined in SVG.)

The Workaround

We apply filter: none !important; to JUST ie9.

This div needs a gradient and rounded corners, and actually gets both in IE9.
(Gradient applied via SVG in IE9, because we told IE9 to ignore the -ms-filter.)

To my mild shock, this dead-simple, drop-in technique actually works!

Here’s hoping the final IE9 build either doesn’t support -ms-filter anymore, or that this fix still works.
