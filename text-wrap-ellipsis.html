Test 1

Test element with word-wrap: nowrap; as well as text-overflow:ellipsis;

Test 2

Test element with maximum dimensions (max-height and explicit width) but no "nowrap" property. Also has text-overflow:ellipsis;

Test 3

short short short loooooooooooooooooooong short short looooooooooong short short

Explanation

The first element has word-wrap:no-wrap; as well as a set width and overflow: hidden. This is a textbook case of when text-overflow: ellipsis should be visible (works in IE6+, Webkit, and Opera, but not yet Firefox a/o FF4b9).

The second element is styled exactly the same except that it omits any word-wrap declaration at all, but it has a max-height of 2.4em. So text is definitely being clipped, but you can’t see it in my browser (Chrome 8 / OS X), despite the current draft of CSS3-UI implying that there is at least a good chance that text-overflow should be invoked (emphasis mine):
This property specifies the behavior when text overflows a block-level element that has ‘overflow’ other than ‘visible’. Text may overflow for example when it is prevented from wrapping (e.g. due to ‘white-space:nowrap’ or a single word is too long to fit).


The third test is a test to see whether more than one word that is too long can get the ellipsis treatment.
