Print stylesheet test

This page exists to test the claim that Firefox will ignore @media print styles unless the stylesheet was loaded with a <link> tag media value explicitly including "print" — and that the "all" keyword is not good enough.

According to my experiments printing this page (to PDF! I’m not a monster), on 2018-10-23, this is not true. Both Chrome and Firefox apply all print styles, regardless of inclusion mechanism.

Is the main stylesheet loaded? It should always be loaded. It is included with <link media="all">.

Are the main stylesheet @media print styles applied? This should always be the case on print. They are defined with @media print inside a stylesheet loaded with <link media="all">.

Has the print stylesheet applied? This should always be the case on print. It is included with <link media="print">.

Are the test stylesheet @media print styles applied? This should always be the case on print. They are defined with @media print inside a stylesheet loaded with <link media="screen, projection, print">. (Note: 'projection' is a deprecated media type now.)
