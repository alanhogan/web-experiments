Backticks to Escape Attribute Values

If you browser supports using backticks to support escaping attribute values, the button below should trigger an alert dialog when clicked.

This looks like this: <button onclick=`alert( 'single quotes '+"and double quotes within the attribute value" );`> …

This has implications for XSS (cross-site scripting) attacks and prevention. Not all XSS prevention/scrubbing/sanitization libraries treat backticks specially, and many if not most HTML-escape routines fail to escape backticks.

My understanding of the HTML specification is that backticks are not valid delimiters and should be treated like any other normal character (meaning, not one of < > " ') in an HTML document.
